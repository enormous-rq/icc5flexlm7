Q12. End-User Options File

Please refer to the End-User manual for a general discussion of the End-User Options file. Only exceptional are answered here.

Q12.1 How do I enable the Options file?

Add a path to your Options file to the DAEMON line in the license file:

Before: 

DAEMON xyzd ../daemon_path

After: 

DAEMON xyzd ../daemon_path ../options_file_path
Refer to END USER MANUAL.

Q12.2 Why doesn't TIMEOUT work?

TIMEOUT only works if the application developer specifically supports it. By default, it doesn't work. Often, it's a difficult feature to support, since the application has to determine when it's `idle', and this is sometimes non-trivial to determine.

Q12.3 Is the Options GROUP the same as a Unix group (/etc/groups)?

No. GROUPs are defined in the Options file, and are unrelated to Unix groups.

Q12.4 I need to have more users in a GROUP - how can I get around the 240 character limitation?

Upgrade to v4+ vendor daemons. They support '\' line continuation character, and a GROUP line of up to 2K. Also use multiple GROUP lines with the same groupname concatenated.

Q12.5 I need a GROUP of hosts, not users.

Upgrade to v4+ vendor daemons, where HOST_GROUP is supported.

Q12.6 What is REPORTLOG for?

It's used by FLEXadmin to give an accurate report of historical usage. It's not intended for any other use. This can be enable in the Options File: REPORTLOG filename.
